# V2 統合方針

この `v2/` は、既存の `index.html` を直接継ぎ足すための見本ではなく、**次の本体再構成の土台**として扱う。

## 1. 先に残すもの

既存本体から残すものは次の4つです。

- 認証導線
  - `POST /api/auth/login`
- 課金導線
  - `POST /api/stripe/checkout`
  - `POST /api/stripe/portal`
  - `GET /api/stripe/session`
- 体験ログ連携
  - `POST /api/sheets/add`
- AI会話API
  - `POST /api/chat`

この4系統は、まず**壊さず残す**ことを優先します。

## 2. 先に差し替えるもの

V2から本体へ移す対象は次の5ブロックです。

- ホーム
  - 何を先にやるかが分かるトップ
- 10回講座
  - GROW寄りの基礎コーチング講座
- ケース練習
  - 1on1会話の返し方を試す実践
- 面談前AIコーチ
  - 本番前に Goal / Reality / Options / Will を整理する導線
- ふり返りとダッシュボード
  - 面談後メモと弱点の見える化

## 3. 実装の考え方

本体へ入れるときの原則は次の通りです。

- 既存の認証と課金は流用する
- `v2/` の体験部分だけを本体の中身として置き換える
- 面談前AIコーチは、初期段階ではローカルロジックのまま使う
- APIコストがかかるのは、まず「部下役会話」と「フィードバック」だけに絞る

これは中小企業向け販売を考えると妥当です。
理由は、毎回の準備メモまでAPI化すると、費用と不安定要因が増えるためです。

## 4. 移植順

### Phase 1: 画面骨格の差し替え

目的:
- 本体の見た目と導線を V2 ベースへ置き換える

作業:
- 既存 `index.html` のタブ構成を V2 構成に合わせる
- 既存のログインゲート、トライアル表示、課金表示は残す
- `v2/styles.css` を本体のデザイン基準として移す

完了条件:
- ログイン後に V2 のホーム、講座、ケース練習、面談前AIコーチ、ふり返り、ダッシュボードが見える
- 課金やトライアル表示が消えていない

### Phase 2: 講座・ケース・面談前整理のローカル実装を移す

目的:
- V2 の学習体験をそのまま本体に入れる

作業:
- `v2/data.js` の講座データとケースデータを本体へ移す
- `v2/app.js` のうち、次のローカル処理を先に移す
  - 講座進行
  - ケース選択
  - 面談前AIコーチ
  - ふり返り保存
  - ダッシュボード集計

完了条件:
- APIを呼ばなくても、講座・準備・ふり返りの流れが動く
- `localStorage` ベースで学習ループが回る

### Phase 3: ケース練習だけ既存AI APIに接続する

目的:
- V2の実践体験と既存AIロールプレイをつなぐ

作業:
- 既存 `POST /api/chat` の `action=chat` を継続利用する
- 既存 `POST /api/chat` の `action=feedback` を継続利用する
- V2側のケースUIから、この2つを呼ぶように合わせる

完了条件:
- ケース練習で部下役との会話が続く
- セッション終了後にフィードバックが返る

### Phase 4: 文言とスコアの統一

目的:
- 体験全体を「1on1品質を標準化する訓練SaaS」として整える

作業:
- 講座、ケース練習、面談前AIコーチの言葉を統一する
- スコア表現を次の4軸に揃える
  - 共感
  - 受け止め
  - 深掘り
  - 安心感
- 本番前メモと本番後ふり返りのつながりを強める

完了条件:
- 画面ごとに言い方がズレない
- 初級から実践まで一本の流れで見える

### Phase 5: 企業向け機能の入口だけ足す

目的:
- 将来の法人販売に耐える構造へ寄せる

作業:
- 管理者向け入口を仮置きする
- 受講者一覧、実施回数、弱点傾向の置き場所を決める
- 個人学習画面と法人管理画面を混ぜないようにする

完了条件:
- 今は未実装でも、法人版の拡張先が決まっている

## 5. いまの判断

現時点では、次の判断で進めるのが最も安いです。

- 面談前AIコーチはローカルロジックで残す
- ケース練習のみ OpenAI API を使う
- まずは個人画面の完成度を上げる
- そのあと法人管理を別レイヤーで足す

## 6. 次に着手するファイル

本体移植の最初の対象は次です。

- `index.html`
  - 画面構成の差し替え
- `api/chat.js`
  - V2ケース練習との整合確認
- 新規分割候補
  - `v2/data.js`
  - `v2/app.js`
  - `v2/styles.css`

## 7. この計画のゴール

最終的には、次の一本の流れにする。

1. 10回講座で基礎を学ぶ
2. ケース練習で返し方を試す
3. 面談前AIコーチで本番前に整理する
4. 本番後ふり返りで次回に引き継ぐ
5. ダッシュボードで弱点を見る

この流れが通ると、単なる練習アプリではなく、**1on1の質を底上げする実践ツール**として売りやすくなる。
